# مخططات Mermaid للرسالة (منصة مساعد المعلم)

## 1) مخطط سياق النظام

```mermaid
flowchart LR
  subgraph أصحاب_المصلحة["أصحاب المصلحة والخدمات الخارجية"]
    M["المعلم"]
    A["المسؤول"]
    I["المؤسسة التعليمية"]
  end
  S(("منصة مساعد المعلم"))
  DB[("Turso / libSQL")]
  LLM["Groq LLM"]
  EXP["Chromium / التصدير"]
  M -->|إدارة المنهج والخطط والواجبات والاختبارات| S
  A -->|الإشراف والإحصاءات وإدارة المستخدمين| S
  I -->|السياسات والسياق التنظيمي| M
  I -.->|متطلبات الاستخدام| S
  S <-->|قراءة وكتابة البيانات| DB
  S <-->|توليد المحتوى بالذكاء الاصطناعي| LLM
  S <-->|توليد PDF/Word| EXP
```

## 2) مخطط تفاعل أصحاب المصلحة

```mermaid
flowchart TB
  subgraph معلم["المعلم"]
    M1["تسجيل الدخول"]
    M2["إدارة الصفوف والمواد والوحدات والدروس"]
    M3["توليد وتحرير خطط الدروس"]
    M4["الواجبات والاختبارات"]
    M5["طلبات التحسين والموافقة"]
    M6["التصدير والإعدادات"]
  end
  subgraph مسؤول["المسؤول"]
    A1["لوحة الإحصاءات"]
    A2["إدارة المعلمين والمناهج"]
    A3["المراجعة والتدقيق"]
  end
  subgraph نظام["المنصة"]
    P["وظائف النظام الموحدة"]
  end
  M1 & M2 & M3 & M4 & M5 & M6 --> P
  A1 & A2 & A3 --> P
```

## 3) البنية المعمارية عالية المستوى

```mermaid
flowchart LR

```

## 4) تدفق البيانات الرئيسي

```mermaid
flowchart TD
  U["إدخال المستخدم"] --> F["الواجهة الأمامية"]
  F -->|طلبات مصادقة وموارد| API["واجهة برمجة التطبيقات"]
  API --> V["التحقق ومعالجة المنطق"]
  V -->|استعلامات| DB["التخزين الدائم"]
  V -->|مسودات JSON| AI["نموذج لغوي خارجي"]
  AI -->|مخرجات منظمة| V
  V --> DB
  DB -->|نتائج| API
  API -->|استجابات| F
  F --> O["عرض أو تنزيل المخرجات"]
```

## 5) مسار الاستخدام الرئيسي (رحلة المستخدم)

```mermaid
flowchart TD
  S1(["بدء الجلسة"]) --> S2["تسجيل الدخول"]
  S2 --> S3["تهيئة الملف والإعدادات"]
  S3 --> S4["إنشاء أو اختيار صف ومادة"]
  S4 --> S5["تعريف وحدة ودرس"]
  S5 --> S6["توليد خطة الدرس بالذكاء الاصطناعي"]
  S6 --> S7["مراجعة التحقق والتحرير"]
  S7 --> S8["حفظ الخطة وربطها بالدرس"]
  S8 --> S9{"واجبات أو اختبار؟"}
  S9 -->|نعم| S10["إنشاء واجبات أو اختبار"]
  S9 -->|لا| S11["تصدير أو إنهاء"]
  S10 --> S11
```

## 6) مخطط تسلسل: توليد خطة الدرس

```mermaid
sequenceDiagram
  participant U as المعلم
  participant FE as الواجهة الأمامية
  participant BE as الخادم الخلفي
  participant DB as قاعدة البيانات
  participant AI as Groq
  U->>FE: طلب توليد خطة درس
  FE->>BE: POST /api/generate-plan + JWT
  BE->>BE: التحقق من الرمز والمدخلات
  BE->>AI: طلب توليد JSON للخطة
  AI-->>BE: مسودة الخطة
  BE->>BE: التحقق من الصيغة والقواعد
  BE->>DB: INSERT خطة الدرس
  DB-->>BE: مُعرّف السجل
  BE-->>FE: 201 + بيانات الخطة
  FE-->>U: عرض الخطة المحفوظة
```

## 9) تدفق الإدارة والإشراف

